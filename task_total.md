## Итоговые задания спринта

Ниже приведён перечень задач, результат выполнения которых необходимо отправить на ревью по итогам текущего спринта.

### Перечень задач

1. **Развёртывание ELK**
   - Развернуть стек ELK (Elasticsearch, Logstash, Kibana).
   - Подключить ELK ко всем сервисам проекта.
   - Убедиться в корректном сборе и отображении логов.

2. **Исследование выбора хранилища**
   - Провести сравнительное исследование хранилищ данных:
     - MongoDB;
     - альтернативное реляционное хранилище (например, PostgreSQL).
   - Наполнить базы данных объёмом **10+ млн записей**.
   - Провести нагрузочное тестирование:
     - чтение;
     - запись;
     - типовые бизнес-сценарии.
   - Описать результаты исследования и выводы в файле **README.md**.

3. **Разработка API**
   - Создать API, реализующее CRUD-операции для:
     - закладок пользователей;
     - лайков;
     - рецензий.
   - Использовать **MongoDB** в качестве хранилища данных.

4. **Интеграция Sentry**
   - Развернуть Sentry.
   - Подключить Sentry ко всем сервисам.
   - Проверить корректную отправку ошибок и событий.

5. **Настройка CI (GitHub Actions)**
   - Написать конфигурационные файлы CI.
   - Обязательно предусмотреть:
     - запуск линтеров;
     - запуск тестов;
     - использование `strategy` (matrix) для нескольких версий Python;
     - отправку сообщения в Telegram при успешном прохождении CI.

---

Результаты выполнения всех перечисленных задач должны быть оформлены, задокументированы и отправлены на ревью.
