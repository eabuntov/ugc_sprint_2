## Настройка CI

Для автоматизации интеграции кода и запуска сопутствующих проверок необходимо настроить CI-процесс, который покрывает следующий набор задач.

### Требования к CI

1. **Исправление проверки линтерами**
   - Заменить `flake8` на `wemake-python-styleguide`.
   - Устранить все ошибки и предупреждения, выявленные линтерами.

2. **HTML-отчёт по результатам линтинга**
   - Настроить вывод результатов проверки в виде HTML-файла.
   - Использовать встроенные возможности линтеров для генерации отчётов.
   - Пример настройки можно найти в документации линтеров.

3. **Проверка типов**
   - Добавить статическую проверку типов с использованием `mypy`.

4. **Проверка совместимости с версиями Python**
   - Настроить запуск CI для следующих версий Python:
     - Python 3.10
     - Python 3.11
     - Python 3.12
   - Использовать `strategy` (matrix) для параллельного выполнения проверок.

5. **Уведомления в Telegram**
   - Добавить отправку сообщения в Telegram при успешном прохождении CI.
   - Уведомление должно отправляться только при полном успехе всех этапов.

6. **Триггеры запуска CI**
   - Настроить автоматический запуск CI-процесса при создании новых pull request в ветку `master`.

7. **Качество кода**
   - Исправить все недочёты, выявленные:
     - линтерами;
     - статическими анализаторами;
     - проверкой типов.

---

В результате CI должен обеспечивать стабильную проверку качества кода, его типобезопасности и совместимости с поддерживаемыми версиями Python, а также информировать команду о статусе проверок.
